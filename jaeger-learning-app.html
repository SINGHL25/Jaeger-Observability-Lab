What is Distributed Tracing?

When a user clicks "Buy Now," their request might touch 12 different microservices — auth, inventory, pricing, payment, notification... How do you know which one is slow?

🚚 REAL-WORLD ANALOGY

Like a courier tracking system — every package gets a tracking number (Trace ID), and each warehouse scan is a Span. You see the entire journey, including delays.

🎯
Why Jaeger?

Jaeger is a CNCF graduated open-source distributed tracing platform, born at Uber. It implements the OpenTelemetry standard.

Open Source CNCF Graduated OpenTelemetry Kubernetes Native

Core Concepts
Master the vocabulary and mental models before diving into the architecture.
🔗
Trace

A Trace is the complete story of a single request as it travels through your system. It has a unique Trace ID (e.g., 4bf92f3577b34da6a3ce929d0e0e4736).

📦 ANALOGY

The courier tracking order — one order number covers the package's entire journey across all warehouses.

TRACE STRUCTURE
// A trace contains many spans Trace { traceID: "4bf92f3577b34da6a", spans: [...array of spans...], processes: { /* service metadata */ } }
📊
Span

A Span represents a single operation (e.g., an HTTP call, DB query). It contains a start time, duration, tags, and logs. Spans form a tree via parent-child relationships.

📦 ANALOGY

One scan at a warehouse checkpoint — time stamped, location recorded, duration noted.

SPAN STRUCTURE
Span { traceID: "4bf92f3577b34da6a", spanID: "a2fb4a1d1a96d312", parentID: "null", // root span operation: "HTTP GET /checkout", startTime: 1717000000000, duration: 142, // microseconds tags: { "http.status_code": 200 } }
📡
Context Propagation

The Trace ID must be passed forward every time one service calls another. This is done via HTTP headers or gRPC metadata.

HTTP HEADERS
# B3 propagation format X-B3-TraceId: 4bf92f3577b34da6 X-B3-SpanId: a2fb4a1d1a96d312 X-B3-Sampled: 1 # W3C TraceContext (preferred) traceparent: 00-4bf92f...da6-a2fb4a...12-01
🎲
Sampling

You can't store every trace at scale. Sampling decides which requests to record.

Const 100% or 0% — all or nothing
Probabilistic e.g., trace 1% of requests
Rate Limiting Max N traces per second
Adaptive Jaeger adjusts sampling dynamically
🌊
Full Request Flow — Step by Step
User makes request → API Gateway receives it, creates a Trace ID and root Span
Gateway calls Auth Service → passes Trace ID in HTTP header; Auth creates a child span
Auth calls User DB → another child span records the SQL query + duration
Gateway calls Order Service → new child span, Order calls Inventory → grandchild span
Each service's Jaeger Agent (sidecar) receives spans via UDP and forwards to Collector
Collector validates, indexes, and stores all spans in Elasticsearch
Jaeger UI / Grafana queries Elasticsearch — you see the full trace waterfall timeline
DEEP DIVE
Jaeger Architecture
Production-grade deployment on Kubernetes with all components explained.
🔷
Jaeger Agent

Lightweight sidecar/DaemonSet process that runs alongside each service. Receives spans via UDP:6831 (Thrift compact) locally. Batches and forwards to Collector over TCP.

K8s DAEMONSET SNIPPET
kind: DaemonSet metadata: name: jaeger-agent spec: template: spec: containers: - name: jaeger-agent image: jaegertracing/jaeger-agent:1.53 args: - --reporter.grpc.host-port=jaeger-collector:14250 ports: - containerPort: 6831 protocol: UDP # Thrift compact
🔥
Jaeger Collector

Stateless component that receives spans from Agents. Runs as a Kubernetes Deployment with horizontal scaling. Validates span integrity and writes to the storage backend.

TCP :14267 HTTP :14268 gRPC :14250 OTLP :4317
🔍
Jaeger Query

The Query service hosts the Jaeger UI (:16686) and REST API. It reads from Elasticsearch and lets engineers search, filter, and visualize traces.

🏪 ANALOGY

The customer-facing website of the courier — where you type your tracking number and see the full journey map.

🗄️
Storage — Elasticsearch

Jaeger uses Elasticsearch as its production storage. Creates one index per day (jaeger-span-YYYY-MM-DD). Index lifecycle management controls retention.

COLLECTOR CONFIG
--es.server-urls=http://elasticsearch:9200 --es.num-shards=5 --es.num-replicas=1 --es.index-prefix=jaeger --es.tags-as-fields.all=true

Integrations
How Jaeger connects with the full observability stack — Kubernetes, Prometheus, Grafana, and Alertmanager.
☸
Kubernetes Integration

Deploy Jaeger using the Jaeger Operator for production K8s deployments. The operator manages lifecycle, upgrades, and configuration.

HELM INSTALL
# Add Jaeger Helm repo helm repo add jaegertracing \ https://jaegertracing.github.io/helm-charts # Install Jaeger Operator helm install jaeger-operator \ jaegertracing/jaeger-operator \ -n observability \ --create-namespace # Deploy a production Jaeger instance kubectl apply -f jaeger-production.yaml
🔴
Prometheus Integration

Jaeger exposes Prometheus metrics at /metrics on each component. Key metrics include span receive rates, storage latency, and queue depths.

KEY METRICS
# Collector metrics jaeger_collector_spans_received_total jaeger_collector_spans_dropped_total jaeger_collector_queue_length jaeger_collector_save_latency_bucket # Query metrics jaeger_query_requests_total jaeger_query_latency_bucket
📊
Grafana Integration

Grafana connects to Jaeger as a data source for trace visualization alongside metrics dashboards. Use Tempo correlation for metrics-to-traces linking.

🎯 USE CASE

Grafana shows "p99 latency spiked at 14:32" → Click the spike → automatically opens the Jaeger traces from that exact time window.

🔔
Alertmanager Integration

Prometheus fires alerts based on Jaeger metrics. Alertmanager routes them to PagerDuty / Slack / OpsGenie.

ALERT RULE EXAMPLE
groups: - name: jaeger.rules rules: - alert: JaegerSpanDropped expr: | rate(jaeger_collector_spans_dropped_total[5m]) > 10 for: 2m annotations: summary: "Jaeger dropping spans!"
📡
OpenTelemetry — The Modern Way ⭐ IMPORTANT

Modern deployments use OpenTelemetry SDK in your code + OpenTelemetry Collector to export to Jaeger. This gives vendor-neutral instrumentation.

Go SDK Java SDK Node.js SDK Python SDK Auto-instrumentation
NODE.JS EXAMPLE
import { NodeSDK } from '@opentelemetry/sdk-node' import { JaegerExporter } from '@opentelemetry/exporter-jaeger' const sdk = new NodeSDK({ traceExporter: new JaegerExporter({ endpoint: 'http://jaeger:14268/api/traces' }), serviceName: 'order-service' }) sdk.start()
